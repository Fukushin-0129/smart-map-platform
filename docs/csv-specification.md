# CSV Specification

## 対応形式

- CSV インポート。
- CSV エクスポート。
- 日本語 UTF-8 BOM 対応。
- 列マッピング。
- 先頭に説明行がある CSV への対応。
- 緯度経度が1セルに入っている CSV への対応。
- 重複判定。
- エラー行表示。
- インポート結果確認。

## CSV インポートの流れ

1. ファイル選択。
2. 文字コード判定。
3. ヘッダー行候補検出。
4. 列マッピング確認。
5. プレビュー表示。
6. エラー行・警告行の表示。
7. 重複候補の表示。
8. インポート実行。
9. 結果サマリー表示。

## 標準列

| 標準項目 | 列名候補 |
| --- | --- |
| name | 名前、店舗名、物件名、施設名、Place Name |
| address | 住所、所在地、Address |
| latitude | 緯度、lat、latitude |
| longitude | 経度、lng、longitude |
| category | カテゴリ、分類、種別 |
| status | 状態、ステータス、配布状況 |
| assignee | 担当者、担当 |
| note | メモ、備考、説明 |
| tags | タグ、Tags |
| phone | 電話番号、TEL |
| website | URL、Web、Website |

## エクスポート

- UTF-8 BOM 付き CSV を標準にする。
- 表示中の Project / Layer / Filter を反映するか、全件出力するか選べる。
- History を含めるか選べる。
- 写真は URL または写真 ID を出力する。Data URL の直接出力は避ける。

## 重複判定

- `externalPlaceId` が一致。
- 正規化した `name + address` が一致。
- 緯度経度が一定距離以内で名称が類似。
- チラシ配布では `no + name`、または `name + lat/lng` を優先。

## 現状との差分

現状はチラシ配布 CSV に特化し、見出し行自動検出と配布実績 CSV 出力を実装しています。将来は汎用列マッピング、エラー行管理、プレビュー、複数テンプレート対応へ拡張します。
