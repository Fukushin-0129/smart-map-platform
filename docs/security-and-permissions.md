# Security and Permissions

## 基本方針

地図データは住所、訪問履歴、写真、担当者、営業先など機微情報を含む可能性があります。販売に向けて、認証、認可、監査、データ分離を初期設計から組み込みます。

## 公開範囲

- 非公開: オーナーと招待メンバーのみ。
- 招待者のみ: 招待済みユーザーだけがアクセス可能。
- リンクを知っている人: リンク保持者が閲覧可能。編集は別権限。
- 一般公開: 検索や公開ページで閲覧可能。

## 権限

### オーナー

- Project 設定変更。
- メンバー管理。
- 権限変更。
- データ削除。
- エクスポート。
- 課金設定。

### 編集者

- Place 作成・更新。
- Knowledge 追加。
- 写真追加。
- CSV インポート。
- 許可された範囲のエクスポート。

### 閲覧者

- Place 閲覧。
- Knowledge 閲覧。
- 許可された公開範囲の地図閲覧。

## 監査ログ

History に次を記録します。

- 操作日時。
- 操作者。
- 対象 Project / Place。
- 操作種別。
- 変更前後の差分。
- インポート元。
- AI 実行結果の採用有無。

## Supabase 移行時の要件

- Supabase Auth を利用する。
- Project ごとに Row Level Security を設定する。
- 写真は Supabase Storage に保存する。
- public bucket と private bucket を分ける。
- 招待リンクには期限と取り消し機能を持たせる。

## localStorage 期間の注意

- localStorage は端末内保存であり、チーム共有や厳密な権限管理には使わない。
- 共有機能の導入前に、既存 localStorage データをユーザー確認付きでクラウドへ移行する。
- 移行に失敗した場合に備え、JSON バックアップを提供する。

## API キー

- Google Maps API キーは HTTP リファラー制限を必須にする。
- Vercel 環境変数で管理する。
- クライアント公開キーで利用できる API のみに限定する。
- 課金 API は利用量上限と監視を設定する。
